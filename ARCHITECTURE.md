# Architecture Rules

This document defines the architectural invariants of the Verso project.

These rules should rarely change.

---

## Rule 01

Every new capability starts with a documented problem.

---

## Rule 02

Every problem must produce at least one Use Case.

---

## Rule 03

Every architectural decision must be documented before implementation.

---

## Rule 04

Every implementation must follow an accepted specification.

---

## Rule 05

Examples validate architecture.

They never define it.

---

## Rule 06

The repository should continuously reduce complexity.

Never increase it without justification.
