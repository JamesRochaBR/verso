# RFC-0000

## Title

RFC Process

---

## Status

Draft

---

## Purpose

This document defines how architectural decisions are proposed, discussed and incorporated into the Verso framework.

Every significant architectural change must be documented as an RFC before implementation.

---

## RFC Lifecycle

Draft

↓

Review

↓

Accepted

↓

Implemented

↓

Stable

---

## Rules

1. Every RFC receives a unique sequential number.

2. RFC numbers are never reused.

3. Accepted RFCs become part of the official specification.

4. Implementations must follow the accepted RFC.

5. Existing RFCs may only change through a new RFC.

---

## Philosophy

Architecture evolves through specifications.

Code follows architecture.

Never the opposite.
